# Contributing

`some-engineer-mind`은 비전공 개발자를 위한 Claude Code 마인드셋 스킬 컬렉션입니다. 새 스킬·개선·번역 PR을 환영합니다.

## 새 스킬 추가하기

1. 폴더 이름은 `kebab-case`. 한 문장으로 그 스킬이 *무엇을 막는지* 드러나야 함.
2. 폴더 안에 `SKILL.md` 하나만. 다음 frontmatter 필드를 반드시 채울 것:
   ```yaml
   ---
   name: <폴더 이름과 동일>
   description: "Triggers on <영문 트리거 키워드들> + <한 줄 영문 요약>. <한국어 본문 설명>"
   when_to_use: "<영문 한 줄 + 한국어 한 줄>"
   allowed-tools: <Anthropic 공식 스펙의 도구 이름들, 공백 구분>
   ---
   ```
3. 본문 섹션은 4개 고정:
   - `## Why this mindset` — 이 스킬이 말하는 것 (한국어 2-3 문단)
   - `## Instructions` — Claude가 적용하는 방식 (한국어, 5단계)
   - `## Examples` — 동작 예시 (한국어, 2-4개 시나리오)
   - `## Pitfalls` — 막는 흔한 함정 (한국어, 5-7개 bullet)
4. description + when_to_use 합계 **1,536자 이내** (Anthropic 매칭 한도).
5. 비공식 필드(`auto_invoke` 등) 추가 금지.

## 톤

- "옳고 그름"보다 *"왜 이 자리에 멈춰야 하는가"*.
- 명령형보다 *"이렇게 하면 가볍다"*.
- 비전공자/비개발자도 읽을 수 있어야 함. 전문용어는 풀어 쓰기.

## PR 체크리스트

- [ ] frontmatter 4개 필드 모두 채움
- [ ] 한국어 본문 4개 섹션 구조
- [ ] description 맨 앞 100자 안에 영문 트리거 키워드
- [ ] 기존 스킬 11개와 톤/길이 유사
- [ ] `node bin/install.js --dry-run` 실행 시 새 스킬이 리스트에 보임 (인스톨러 `SKILS` 배열에 추가했는지 확인)

## 번역·교정

- 오타/표현 개선 PR은 본문만 수정하고 frontmatter는 건드리지 않습니다.
- 영문 풀버전 번역 PR은 별도 폴더(예: `en/`)로 분리 — 메인은 한국어 우선 유지.

## 행동 규범

비전공자를 위축시키지 않는 글쓰기. *"이건 좀 기술적인 거라서요"* 같은 표현 금지. 이 컬렉션의 가치는 *읽는 사람이 듣는 사람으로 남게 하지 않는 데* 있습니다.
